What is Sebca?

Search Engine Biased Color Analysis (SEBCA) is a simple online tool that connects words with colors.



Type in a phrase in the text box that you would like to have a relevant color pallete for- hit GO! and you will have an array of colors associated with the phrase.


How does Sebca work?

To put it simply, Sebca uses image search with the relevant tag to gather information about the dominant colors of the relevant search

First: Sebca uses Microsoft Cognitive Service API to gather relevant image urls through Bing Images Search with the keyword user provides and gets back a number of urls (Currently at 9-urls)



Second: after reciving the relevant urls Sebca algorithm reads the images and put them into one image in a grid form and analyzes them for dominant colors by using histograms for color groups.



Third: after finding the dominant colors it displays the colors with the relevant tag to the user and puts them in a database.



Some of the important problems with Sebca!


The Zebra Problem

One of the first and main problems faced at early stages of testing was the overwelming presence of whites, and blacks in images. Images that were in clip art had a lot of white around them and real photos had usually had a dark gray and black dominance in them

The human brain is good at filtering out irrelevant colors and seeing color despite higlighted or shaded areas. Sebca algorith was not able to do this, so currently a spectrum of gray including back and white are excluded in the outputs. this has solved most of the problems with searches, except the one where user expects to see either black or white, or both, in the output. An obviouse example of this could be found in searches such as Zebra.



You can see that Sebca does well when all the RGB spectrum is included for the Searchword Zebra, but excluding the grays would result in the third palette which lack the two main colors thet the user expects to see. Instead it has included some greens from the images' background.

Even with such a problem it is better to keep whites and blacks out of the output since most of users do not search for phrases that include white or black as main color, yet most results yeild black and white as dominant colors.


The Flamingo Problem

The Flamingo problem is the problem of the algorith not being ale distinguish the foreground/subject from the background/environment. Although the presence of colors from the environment is sometimes helpful in giving a better picture of a keyword, it, in some ocassions, overwelms the actuals subject. A good example of this problem could be seen with the search word 'flamingo'.



One solution

To correct this problem, now the algorithm takes more samples as it gets closer to the center of the image, since statistically speaking it is more likeley that the subject of an image is placed in the center. With this assumption Sebca results in much more center focused color outputs.


A simplified grid of analysis that Sebca currently uses for 9 images. Each number represents an image that has comethrough the API request for images. You can see that each image is resized and placed in the grid and sampled with favouring the center of each image.
